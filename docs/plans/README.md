# Plans

Implementation plans for features and changes.

## Format

Plans use the naming convention: `YYYY-MM-DD-<topic>-plan.md` or `YYYY-MM-DD-<topic>-design.md`

## Plan Template

```markdown
# Plan: Title

## Overview
Brief description of what we're building.

## Goals
1. Goal 1
2. Goal 2

## Design
Architecture and approach.

## Implementation Steps
1. Step 1
2. Step 2

## Testing
How to verify it works.
```

## Workflow

1. Create plan using brainstorming skill
2. Get user approval
3. Execute plan (optionally in git worktree)
4. Mark plan as completed
